# Archived Eval Reports
DevDen · Historical efficiency reports

## Purpose

Archived reports are retained for:
- **Trend analysis**: are agents improving, stable, or degrading over time?
- **Recommendation tracking**: did CPE act on prior recommendations? Did the changes help?
- **Pattern recognition**: are the same issues recurring across multiple cycles?

## Archival Convention

Move a report from `active/` to `archived/` after:
- CPE has acknowledged and responded to the report
- The recommended actions have been decided (acted on or declined with rationale)

## Using Archived Reports for Trend Analysis

At the start of each eval cycle, check the last 2–3 archived reports for each agent.

Key questions:
- What was the score trend? Is it improving?
- Were prior recommendations implemented? Did they produce improvement?
- Are there persistent issues that have not been resolved? Escalate.

If a score dimension has been ≤ 2 for three consecutive cycles without improvement, it indicates the recommendation wasn't effective. Propose a different approach in the next report.
